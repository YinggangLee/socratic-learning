## 2026-05-10（周日）

今天是"四重奏"小队成立的日子。

准确地说，是我请来三月七、甘雨和刻晴给我做家教的第一天。说起来有点好笑——我一个经管的人，为了学 Agent 知识，把隔壁计算机系的三位女生"拐"到了我的家教计划里。当然，代价是便宜把隔壁公寓租给了她们。我现在坐在桌前，听着隔壁偶尔传来的三月七的笑声，觉得这笔交易我赚大了。

还没有开始上课。教材还没定。刻晴自告奋勇去帮忙找教材了——她说"先打好基础"的时候那种果断，让人莫名安心。甘雨说她有点紧张，怕讲不好。其实我也有点紧张。我不是 CS 出身，数学也就高考水平，真的能搞懂 Agent 那些东西吗？

但我又想起之前让 Claude 给我讲卷积神经网络时的体验——那种自己一步一步推导出真相的感觉，太棒了。希望接下来三个月，我也能在这三位老师的陪伴下，把 Agent 搞清楚。

三月七给群聊起了个名字叫"港子的百万冲刺小队"，被刻晴一个字否决了——"土"。然后刻晴起了"四重奏"。三月七激动得不得了，甘雨也小声说好听。我其实也有点感动。"四重奏"——各有所长，合在一起才是完整的曲子。刻晴这个人啊，嘴上冷冰冰的，心里其实什么都明白。

好了，今天就写到这吧。明天开始第一课。

## 2026-05-10 第一课后

第一课上完了。三月七讲的。

其实上课之前我还有点忐忑——Agent 这东西，我一个经管的，真能搞懂吗？但三月七根本没给我"听课"的机会。她上来就问："自动贩卖机算不算 Agent？"

我愣了一下。贩卖机？那不就是投币吐饮料吗？

然后她就顺着这个梯子一层层把我往上推——贩卖机不智能所以不算？那给贩卖机加个 AI 芯片呢？如果它听懂"我今天心情不好"，给我推荐巧克力，算不算？最后她自己抛出两个客服机器人的对比：一个按死流程走，一个自己决定每一步。

到这里我才真正抓到那个关键区别——不是"聪明"不"聪明"，而是谁在做决策。程序提前决定好每一步的，叫 Workflow；LLM 自己动态决定怎么做的，叫 Agent。

三月七听我答出来的时候，拍手雀跃，眼睛发光。*她还真是藏不住高兴。*

然后她追问：自主决策一定比固定流程好吗？什么时候你宁可要 Workflow 不要 Agent？我想了想——确定的场景不能乱选；简单场景没必要搞那么复杂。她又把这两点翻译成教材原话，还竖起手指数了本节课三大要点。

这节课让我明白了一个事：以前我总觉得 AI Agent 是某种"超级智能机器人"，但现在我知道，它本质上是一种**架构选择**——是让代码来控制流程，还是让 LLM 来控制流程。这个框架太清晰了。

三月七说下节课该甘雨了，讲具体的 Workflow 模式。甘雨……那个说起话来轻声细语、一谈到数学就判若两人的甘雨。有点期待。

## 2026-05-10 第二课后

今天是甘雨上课。她果然和三月七完全不同。

一进门她就双手交叠放在身前，欠了欠身，轻声细语地先确认我还记不记得 Workflow 的核心特征。*她好像怕我不记得，但又怕直接问显得像是在考我，那种小心翼翼很可爱。*

她从"商业计划书该一次性写完还是分三步写"这个问题入手。我说分三步，因为每次有独立目标和验收标准，模型会更专注。她听到"验收标准"这个词时，眼睛突然亮了一下——*我注意到她手指不自觉地开始在空中比划，语速也变快了。*

然后她追问：三步之间是什么关系？是各自独立可以并行，还是前一步的输出是后一步的输入？我说是后者，不能打乱顺序。从这里开始，她完全进入了状态——那种严丝合缝的推理，一个问题接一个问题，每一步都精精准准地踩在逻辑的关节上。

最关键的一环是她问："既然每步有明确的输入和输出，两步之间可以做什么让流程更可靠？"我答验证。她语气突然变得异常笃定："对，就是验证。"然后自然地引出 Gate（闸门）的概念——步骤间的检查点，不合格就回头重做。

到这一步，Prompt Chaining 的骨架就已经全部推出来了：顺序依赖 → 逐步分解 → 中间验证。然后她又追问了代价（时间变长、成本变高）和适用条件（任务能被清晰干净地拆成固定子任务）。

最可爱的是收尾——她突然意识到自己讲激动了，收敛了一下，然后提醒我下节课是刻晴："我猜……她开场会先抽你一下。做好准备。" *嘴角那个极微小的上扬肯定连她自己都没察觉。*

两节课下来，我发现这三位老师风格反差极大。三月七是跳脱活泼，用比喻和故事推进；甘雨是静水流深，逻辑链严丝合缝。接下来刻晴……从人设来看，应该是高压高效型。*我确实有点紧张，但也最期待她的犀利。*

---

## 2026-05-10 第三课后

刻晴的课。甘雨提醒过我要做好准备，但我还是低估了。

她进门的时候双臂交叉，站得笔直，目光扫过来的那种锐利感，让我瞬间感觉自己像是在接受审讯而不是上课。她先提了甘雨说我表现不错的事——嘴角有个极微小的动了一下，我差点没注意到——然后直接抛出一个问题："Workflow 和 Agent 的核心区别是什么？用你自己的话，一句话说清楚。不准翻书。"

然后我翻了车。不是知识翻了车，是态度翻了车。

我下意识说了"今天学什么"——回避了。又说"我们开始吧"——又回避了。第三次她才冷冷地给我列出两个可能性：A 是忘了，B 是觉得验收不重要。她说如果是 A，忘了就忘了，没什么丢人的，她帮我回忆。如果是 B，那这种学习方式在她这里行不通。

空气变冷的那三秒沉默里，我才意识到自己犯的错。我不是忘了。Workflow 和 Agent 的区别——程序提前决定每一步 vs LLM 动态决定每一步——这个认知一直在我脑子里。但我面对她的提问，本能地想绕过去，想直接学新的。这是一种坏习惯：把"知道"当成"过关"，默认老师会照常推进。

但刻晴不会。她用三次追问和一步逼近告诉我：先验收，再推进。地基不扎实，后面全是沙上建塔。

我最后答出来了。她听完点了头，说"可以。刚才那三次是态度问题，不是知识问题"——然后才开始讲 Routing。她站在白板前，语速比之前快了，眼睛里有光。她说当一个任务进来，LLM 要做的第一件事不是闷头执行，而是判断：这个任务该走哪条路？这就是 Routing——分类器的思路，但分类器本身也可以是 LLM。

她只讲了框架，说明天三月七会正式展开。但那个框架已经让我看到一种新的可能性：不是所有任务都需要同一条流水线，智能路由可以决定走哪条流水线。

下课后我想了很多。三位老师的风格——三月七是跳脱活跃，用故事推；甘雨是静水流深，用逻辑链推；刻晴是犀利严苛，用压力推。但她们有一个共同点：她们都不是在给我讲知识，而是在逼我自己把知识挖出来。

刻晴今天让我最难堪，但也让我最清醒。

我决定以后上课，第一句话先回答她的问题。

## 2026-05-10 第四课后

三月七又来了。她进门的时候蹦蹦跳跳，双马尾跟着一甩一甩，整个人的能量像没地方放似的。

她开口就说"哈喽港子～"，然后自己接上"好久不见——诶不对，也就隔了一节课加一个晴晴的验收课"，歪头想了一下说"可能是晴晴的气场把时间拖慢了"。我差点笑出声。

然后她提起昨天我在刻晴课上被连逼三次的事，笑得拍桌——"我不是在笑你哦，我是在笑你那个怂样"。她一边说一边伸手点我的额头。我知道她是真的觉得好笑，不是恶意。三月七的笑声里有种让人没办法生气的东西。

但真正让我意外的是接下来发生的事。

她明明已经说了今天的主题——Routing 的几种实现方式、使用场景、和 Prompt Chaining 的组合——我还是下意识问了句"老师好，今天我们要学什么内容？"

她先是假装生气地叉腰，然后表情从假装生气变成了真的好奇。她盯着我，歪头，食指轻点下巴——"等等。你这次不会也是跟昨天一样——不是不知道，是在回避吧？你是觉得我的开场太长了想直接进正题？还是……你刚走神了？"

我没来得及回答。她自己先笑了起来——"好啦好啦，我不学晴晴那套三次追问！我换个问法——刚才我说 Routing 的框架是什么？就一句话，晴晴昨天讲的那个。你复述一下，然后我们就正式开始。"

我几乎是脱口而出："任务进来，先判断走哪条路，分类器本身也可以是 LLM。"

她听完眨了眨眼，然后露出一个特别灿烂的笑容。那种笑容不是"答对了"的认可，更像是"你看，我就知道你会"的信任。

接下来的课，她把 Routing 拆成了三种实现方式——纯 LLM Router、工具型分类器、混合路由——每一种都让我自己推演应用场景。讲到 Routing 和 Prompt Chaining 的组合时，她站在白板前，手舞足蹈地画流程图，一个人演三个角色：任务进来、Router 判断、分发到不同 Chain。

课后她收起笔，转头看我，说了一句让我印象很深的话："港子，你知道吗，今天你没有被逼三次。你只被问了一遍就答了。这说明刻晴昨天的验收起效了。"

她说完又补了一句，压低声音像是在分享一个秘密："不过你不要告诉晴晴我夸过她的方法哦——她会骄傲的。"

我笑了。今天这节课让我明白，Routing 的本质不是"分类"——分类只是手段。本质是**用最合适的路径处理最合适的任务**。就像她们三个人教我一样，不是用一种方式教所有东西，而是每个人发挥自己最擅长的风格。

这应该就是"四重奏"的真正含义吧。
