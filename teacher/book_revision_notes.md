# 教材改进点

## 2026-05-10 第 4 课（三月七 — Routing 模式）

### 1. Routing 框架定义的切入点可更形象化
- **暴露问题**：刻晴上节课只给了框架一句话（"任务进来先判断走哪条路，分类器本身也可以是 LLM"），三月七开场复述时我立即能答出，但三月七担心学生只是记住而非理解。
- **改进建议**：教材在引入 Routing 时，可增加一个生活化类比（如"医院分诊台"或"快递分拣中心"），帮助学生从直觉出发理解"分类—路由—执行"的流程，而非只记定义。

### 2. Routing 与 Prompt Chaining 的组合逻辑需要更明确的过渡
- **暴露问题**：三月七在课上发现，Routing 和 Prompt Chaining 的衔接处是学生的理解盲区——什么时候先 Route 再 Chain？什么时候 Chain 内部也需要 Route？
- **改进建议**：教材应在 Routing 章节末增设一个"Routing + Chaining 组合模式"小节，用流程图展示两种组合方式，并给出各自适用场景的对比表。
